# Delta Updates

Delta updates can **reduce the amount of data transferred and the time required to install a new version** by reusing locally available parts of the old version of a system. This is especially useful for devices on metered or bandwidth-constrained connections. Delta updates are particularly effective when there are frequent updates with small changes — according to [our benchmarks](/blog/efficient-delta-updates), they can **reduce the amount of data transferred by up to 90%**.

Rugix is the only open-source OTA solution that ships with ready-made support for both **dynamic** and **static** delta updates.

## Dynamic vs. Static

There are two ways to compute the delta between an old and a new version, and Rugix supports both.

**Dynamic delta updates** let Rugix Ctrl decide _on the device_ which parts of the new bundle it already has locally and which it still needs to download. Under the hood, this is HTTP range queries against the bundle, driven by the [block index](./reference/update-bundles#block-encoding) that every Rugix bundle carries. The server only needs to support range requests — almost every HTTP server does, including S3-compatible object stores. Nothing has to be pre-computed.

**Static delta updates** rely on a pre-computed patch between two specific versions. `rugix-bundler delta <old> <new> <patch>` produces a patch bundle that the device installs in place of the full bundle. Static deltas can be roughly twice as efficient as dynamic ones for the same update, but the operational cost is real: you have to compute deltas for every relevant version pair and route the right patch to the right device.

:::tip
**Unless your devices are on a metered or bandwidth-constrained connection, we recommend dynamic delta updates.** They are a quick win — turn them on by enabling block indices and serving bundles over HTTP. Save static deltas for the cases where the additional bandwidth savings actually matter.
:::

## What's Available

Rugix Ctrl ships dynamic and static delta support out of the box. Bundles built by Rugix Bakery have block indices configured by default, so a Bakery setup serving bundles over an HTTP server with range support already gets dynamic deltas for free.

For the operational details — how to enable each, the `rugix-bundler` CLI, server requirements, the slot index commands, and current limitations — see the [Delta Updates reference](./reference/delta-updates).

## Update Simulator

`rugix-bundler simulator` lets you estimate how effective different delta techniques would be on _your_ update history. Drop a sequence of bundles in and see how much each technique would have transferred. See the [reference](./reference/delta-updates#update-simulator) for usage.
